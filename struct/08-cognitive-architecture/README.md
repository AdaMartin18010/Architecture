# 08 认知架构与复用决策

## 定位

软件复用不仅是技术问题，更是认知问题。研究人类开发者在复用决策中的认知过程。

## 核心内容

- **ACT-R** (Adaptive Control of Thought-Rational): 复用意图的表征与模式匹配
- **BDI** (Belief-Desire-Intention): 开发者对复用资产的认知模型
- **双系统理论** (Kahneman): 系统 1（直觉）vs 系统 2（理性）的复用决策差异
- **认知负荷理论** (Sweller): 内在/外在/相关负荷的量化模型
- 专家 vs 新手的复用模式识别差异
- AI 辅助复用决策的认知增强模型（RAG + LLM）
- 认知偏差与复用决策的修正策略（现状偏差、过度自信、沉没成本等 7 种偏差）

## 关键公理
>
> **公理 C.1** (Cognitive Load Conservation): 开发者的认知资源是有限的。复用资产的设计目标应是**降低外在负荷**和**优化相关负荷**，而非消除内在负荷。
