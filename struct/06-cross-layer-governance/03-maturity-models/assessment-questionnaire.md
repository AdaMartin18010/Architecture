# 架构复用成熟度评估问卷

> **版本**: 2026-06-06
> **定位**: 评估组织在架构复用方面的成熟度，覆盖五个等级

---

## 使用说明

本问卷包含 30 个问题，分为 6 个维度。每个问题按 1-5 分评分：

| 分值 | 含义 |
|------|------|
| 1 | 完全没有 |
| 2 | 偶尔/非正式 |
| 3 | 部分定义并执行 |
| 4 | 全面定义并量化 |
| 5 | 持续优化，行业领先 |

---

## 维度 1: 战略与文化 (5 题)

1. 组织是否有明确的复用战略和目标？ [1-5]
2. 高层管理者是否支持并投入资源到复用计划？ [1-5]
3. 复用是否被纳入绩效考核？ [1-5]
4. 团队是否主动寻找和贡献可复用资产？ [1-5]
5. 组织是否有"复用优先"的文化？ [1-5]

## 维度 2: 资产管理 (5 题)

1. 是否有统一的资产目录管理可复用资产？ [1-5]
2. 资产是否按层次（业务/应用/组件/功能）分类？ [1-5]
3. 资产是否有明确的所有者和 SLA？ [1-5]
4. 资产的搜索和发现是否方便？ [1-5]
5. 是否有资产退役和版本管理的流程？ [1-5]

## 维度 3: 治理与流程 (5 题)

1. 是否有复用资产的质量门禁？ [1-5]
2. 是否有复用资产的变更管理流程？ [1-5]
3. 是否有跨层一致性检查机制？ [1-5]
4. 是否有复用价值度量和反馈机制？ [1-5]
5. 是否有安全/合规审查集成到复用流程中？ [1-5]

## 维度 4: 技术与工具 (5 题)

1. 是否有标准化的组件仓库/包管理？ [1-5]
2. 是否有自动化测试覆盖可复用资产？ [1-5]
3. 是否有 SBOM 和安全扫描集成？ [1-5]
4. 是否有 CI/CD 流水线支持资产发布？ [1-5]
5. 是否使用 AI 辅助资产推荐/生成？ [1-5]

## 维度 5: 知识与学习 (5 题)

1. 是否有复用资产的使用文档和示例？ [1-5]
2. 是否有定期的复用最佳实践培训？ [1-5]
3. 是否有 ADR/技术决策记录？ [1-5]
4. 是否有社区或论坛讨论复用问题？ [1-5]
5. 是否有复用成功案例分享机制？ [1-5]

## 维度 6: 价值实现 (5 题)

1. 是否能量化复用带来的开发时间节约？ [1-5]
2. 是否能量化复用带来的缺陷减少？ [1-5]
3. 是否能量化复用对上市时间的影响？ [1-5]
4. 复用资产的采用率是否被追踪？ [1-5]
5. 复用资产的 ROI 是否被定期审查？ [1-5]

---

## 评分与等级

```
总分 = 各维度平均分之和 × 6

等级划分:
├── L1 临时 (Ad-hoc): 总分 < 60
├── L2 管理 (Managed): 60 ≤ 总分 < 90
├── L3 定义 (Defined): 90 ≤ 总分 < 120
├── L4 量化 (Quantitative): 120 ≤ 总分 < 150
└── L5 优化 (Optimizing): 总分 ≥ 150
```

## 改进建议矩阵

| 维度 | 低分时的典型改进项 |
|------|------------------|
| 战略与文化 | 制定复用战略，获得高管支持，纳入考核 |
| 资产管理 | 建立资产目录，定义分类标准，明确所有者 |
| 治理与流程 | 建立质量门禁，变更管理，一致性检查 |
| 技术与工具 | 引入包管理，自动化测试，SBOM，CI/CD |
| 知识与学习 | 完善文档，开展培训，建立社区 |
| 价值实现 | 建立度量体系，追踪采用率，定期 ROI 审查 |
